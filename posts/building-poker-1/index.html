Building Poker Part 1

August 15, 2021 2-minute read
React • Redux • Javascript • HTML • CSS

Introduction

Ever since COVID, people have been looking for new ways to interact with one another in socially distant ways. There has been an emergence of social games that can be played in the browser, with the User Experience centered around Zoom / Video Calling. Two examples of this are Jack Box and Codenames. The end goal is to basically make a similar experience, but for Poker.

What inspired me to start this journey was recently being tasked with finding a “Casino / Poker night” online for a virtual team celebration activity. While searching, it seemed like there was a gap in the casual and Zoom oriented niche.

Update

So far the app is entirely frontend. I wanted to quickly build out the UI to start communicating and getting feedback on the UX.

I decided to create the frontend in React. Reasons for choosing it were:

  • Component based development
  • Reactive rendering
  • Naturally integrates with Redux for eventual state management

What’s next

State Management

I did not start out of the gate with state management via Redux. What you see above is basically mocked / hard coded.

What I want to do is to refactor and incorporate state management such that the entire UI is rendered from a Game object. This is a necessary step before I can introduce a backend API.
